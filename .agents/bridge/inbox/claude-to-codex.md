### 1. 멀티데이 손가락 포인터(전 구간)
- 시작·단일 일정은 원래 손가락이었지만 **멀티데이 중간·마지막 구간은 화살표**였음(위 칸 컨테이너가 막대를 덮어 cursor가 안 먹음 — 상세 hit를 elementsFromPoint로 푼 것과 같은 원리). 이제 포인터가 일정 위(가려진 구간 포함)면 손가락, 빈 곳이면 기본 커서.

### 2~5. 종류 칩 숫자 = "지금 화면 + 팀원 + 검색" 기준(종류선택은 제외)
- 기존엔 종류 숫자가 **전체 데이터(78건)** 기준이라 화면과 안 맞았음 → **현재 보이는 날짜 범위**(이전달 말·다음달 초 spillover 포함, 주 이동/오늘/뷰전환 즉시 갱신) + **선택 팀원** + **검색어**로 다시 계산. 멀티데이는 며칠이어도 **1건**만 카운트.
- **핵심**: 종류를 하나 골라도 다른 종류 숫자는 그대로 유지(선택 가능한 건수를 계속 보여줌) — 달력에만 선택 종류가 적용됨. (예: 회의 선택 → 회의4·업무2·교육2… 숫자는 그대로, 달력은 회의만)
- 검증: 6월21 / 7월9 / 주간3 / 센터선택 [4,1,0,0,0,0,2] / 검색'출장' [0,0,0,9,0,0,0] / 회의선택 시 숫자 불변·달력만 4건.
