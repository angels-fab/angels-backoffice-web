# Next Claude Prompt

> 상태: **사용자 실행 대기.** 아래 지시를 그대로 구현한다.
> 참고 시안: `public/calendar-one-line-demo.html`

## 실행 프롬프트

업무일정과 PC 사이드 메뉴 배지를 다음과 같이 수정해줘.

### 1. 사이드 메뉴 위첨자 배지

- 새글 배지의 `N`과 개선메모 배지의 아이콘은 제거한다.
- 아이폰 알림 배지처럼 색상+숫자만 표시한다.
- 새글: 빨간색 원형 배지
- 개선메모: 노란색 원형 배지
- 두 배지는 메뉴명 바로 뒤 우상단에 나란히 배치한다.
- 크기는 14~16px 정도로 작고 선명하게 한다.
- 0건은 숨기고 99건 이상은 `99+`로 표시한다.
- 기존 메뉴 행 오른쪽 배지는 제거한다.

표시 예:

```text
포털개선요청  [4][3]
빨강 4 = 새글 4건
노랑 3 = 개선메모 3건
```

### 2. 업무일정 한 줄 배열

- 종일·멀티데이 일정도 한 줄 높이를 유지한다.
- 표시 순서는 `구분 아이콘 → 시간 → 장소-목적 → 해당자 원형칩`이다.
- 캘린더에는 기존 데이터인 `장소-목적`을 그대로 표시한다.
- 호버 상세에서만 목적을 제목으로, 장소를 세부정보로 분리한다.
- 국내·국외출장도 기존 `국내/국외(목적지)-목적` 규칙을 적용한다.

### 3. 제목과 해당자 간격

- 해당자를 일정 바의 가장 오른쪽 끝으로 밀지 말고 제목 바로 옆에 붙인다.
- 제목이 짧으면 바로 뒤에 해당자 칩이 따라와야 한다.
- 제목이 길면 제목만 말줄임하고 해당자 칩은 항상 보이게 한다.
- 제목에 `flex: 1`이나 해당자에 `margin-left: auto`를 사용해 빈 공간이 벌어지지 않게 한다.
- 권장 간격은 4~6px이다.
- 멀티데이가 주 단위로 분리되면 각 표시 구간에서도 동일하게 적용한다.

기존 필터, 일정 분류색, 호버 상세, 클릭 동작은 유지한다.
이모지는 사용하지 말고 아이콘은 MUI 개별 경로 import를 사용한다.
수정 후 `npm run type-check`를 실행하고 PC 월간 캘린더에서 짧은 제목·긴 제목·멀티데이 일정을 확인해줘.
