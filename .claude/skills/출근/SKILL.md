# 출근 — 작업 시작 준비

사용자가 작업을 시작할 때, 다른 PC(집↔사무실)에서 한 작업을 받아 바로 코딩할 수 있는 상태로 만든다.
출력은 항상 `가독성` 형식(🔧 회색=실행한 작업 / 💡 파란줄=쉬운 의미 / ✋ 노란=사용자 할 일)을 따른다.

## 순서

1. **위치 확인** — 작업 폴더 `angels-backoffice-web`인지 확인. 아니면 거기로 이동.
2. **최신 코드 받기** — `git pull origin main`.
   - 충돌·에러가 나면 **절대 강제로 덮어쓰지 말고** 사용자에게 그대로 보고하고 멈춘다.
3. **의존성 동기화** — `node_modules`가 없거나 `package-lock.json`이 이번 pull로 바뀌었으면 `npm install`.
4. **개발 서버 실행** — `npm run dev`(백그라운드, 포트 3600). 이미 떠 있으면 생략.
5. **상태 요약** — 방금 받은 커밋(최근 `git log` 몇 개), 현재 브랜치, 접속 주소 `http://localhost:3600`를 가독성 형식으로 보고.

## 주의

- 코드 수정에 들어가기 전 저장소의 `CLAUDE.md` 디자인 규칙(이모지 아이콘 금지·MUI만, 카드 왼쪽 컬러보더 금지 등)을 먼저 읽는다.
- pull로 받은 변경이 크면 무엇이 바뀌었는지 한 줄 요약해 사용자가 맥락을 잡게 한다.
