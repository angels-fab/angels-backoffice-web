# STEP24 — 업무현황 회의 뷰 개편

> 작성: 2026-06-16 · 상태: 구현 완료(프런트 전용), type-check + build 통과, 라이브 dev 검증 완료. **미커밋**.

## 목표
업무현황(`/work`)을 회의 때 바로 같이 보는 화면으로 개선. 핵심은 **진행중 업무를 KPI 바로 아래에서 즉시 펼쳐 보기**.

## 변경된 레이아웃 (위→아래)
1. 페이지 헤더
2. **KPI 카드**(상태별 건수 5타일) — 기존 RatioBar(가로 비율막대) 제거
3. **업무 목록**(구 "전체 업무 목록"을 KPI 바로 아래로 이동) — 상태/구분/담당자 필터 + 검색
4. **Remind**(구 "긴급 업무")
5. ~~담당자 현황~~ (숨김)

## 기본 필터 변경
- 진입 시 기본 상태 탭 = **진행중**(`useState<StatusTab>('inProgress')`).
- 사용자는 KPI 타일/상태 칩으로 전체·완료·보류·취소로 전환 가능(기존 전체 접근 유지). 같은 상태 재클릭 시 전체로 토글.

## 아코디언 동작 (진행중)
- 진행중 탭일 때 목록을 **아코디언**(`TaskAccordion.tsx` 신규)으로 렌더.
- 초기 **모두 펼침**(각 항목 `useState(true)`), 사용자가 개별 접기/펼치기 가능.
- 펼침 내용: 부서/예정/장소/완료 메타 + 업무 내용(taskSubs) + 상세 보기(Drawer)·관련자료 링크 아이콘.
- 그 외 상태 탭은 기존 **컴팩트 1줄 행**(`compactRow`) 유지 — 클릭 시 상세 Drawer.

## UI 정리
- **검토필요 → "검토"**(짧게): KPI 타일 라벨, 목록 상태칩 옆 보라색 필터칩 모두 단축. 필터칩은 상태 칩(…취소) 바로 옆 인라인 배치(긴 가로 영역/구분선 제거).
- **RatioBar 제거**: KPI 타일이 동일 수치를 제공하므로 중복·장식성 가로막대 삭제(회의 뷰 정리).
- **"긴급 업무" → "Remind"**: 사용자 노출 텍스트(섹션 제목). 내부 변수명(`urgent`)은 유지.

## 숨긴 섹션
- **담당자 현황**: `SHOW_MANAGER_STATUS = false` 플래그로 렌더만 숨김. `managers`/`busiest` 집계·JSX 코드는 보존(플래그 true 시 즉시 복원).
