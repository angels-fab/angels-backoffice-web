# STEP 7.6 — 장비운영관리 역할 정리: Asset Dashboard (완료 보고)

> 작업일: 2026-06-14
> STEP7.5에 섞인 프로젝트/구축 성격을 걷어내고, 장비운영관리를 **장비 총괄 현황(Asset Dashboard)**으로 정리. 목표 70→85점.

---

## 왜 (두 시트의 역할 분리)

- **장비운영관리 시트** = "현재 장비 현황"(장비명/분류/담당자/도입금액/상태) → **Asset Dashboard**.
- **장비도입관리 시트** = "구매 절차별 도입 단계/기간"(공고·평가·협상·제작·설치) → **Project Tracking**(STEP8).

STEP7.5는 운영관리에 '도입 단계 비율·도입 우선 장비' 등 **프로젝트 성격이 혼재**되어 도입관리와 겹쳐 보였다. → 프로젝트 정보는 STEP8(도입관리)로 넘기고, 운영관리는 **자산 현황**만 다루도록 정리.

---

## 변경 전 / 후

| 항목 | STEP 7.5 (Before) | STEP 7.6 (After) |
|------|------|------|
| 정체성 | 구축 준비 현황판(프로젝트 혼재) | **Asset Dashboard(장비 총괄 현황)** |
| "도입 단계 비율" RatioBar | 있음 | **제거** (→ STEP8) |
| "도입 우선 장비"(금액순) | 있음 | **제거** (도입 우선순위=프로젝트 정보 → STEP8) |
| KPI 보조문구 | 단계 라벨(구축 준비중 등) | **종 수(자산 관점)** |
| 부제 | FAB 구축 준비 현황 | **장비 총괄 현황 — 전체 자산·상태·담당** |
| 담당자별 현황 | 없음 | **추가** (담당자별 장비 수·상태) |
| 카테고리 현황 | 장비수+예산+상태 | 유지(0인 상태칩 숨김 정리) |
| 전체 목록 탭 | 상태탭 | **탭에 건수 표시** |

**섹션 순서**: ① KPI → ② 카테고리 현황 → ③ 담당자별 현황 → ④ 예산 → ⑤ 전체 목록.
