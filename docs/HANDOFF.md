## 🔄 데모결과 페이지 리뉴얼 — UX 후보 수렴 중 (2026-07-21, 미구현·코드 무변경)

시안만 반복 중이며 **저장소 코드는 아직 안 건드림**. 다른 PC에서 이어가려면 이 섹션 + 아래 시안이면 충분.

**확정된 것**
- 사진 캡션 = **시네마 스크림**(사진 하단 44% 그라데이션 + 제목/설명/장수). 캡션은 `demo_results.photos` JSONB에 `caption` 필드 추가로 구현(마이그레이션 불필요).
- 사진 = **가로 스크롤 캐러셀 + 내부 인덱스(n/4)**, 썸네일 완전 제거. 대표사진=시작 장(cover). PC: 잡고 드래그·가로휠(Shift+휠)·좌우버튼·스크롤바(스크럽+트랙점프) / 모바일: 스와이프. 세로휠은 페이지에 양보(휠 트랩 방지). **동작 JS 검증 완료본**이 `docs/mockups/demo-photo-hscroll.html`에 있음(적대검증 17건 중 must 4건 수정 반영 — dragging 타이머 레이스·휠 트랩·엣지버튼 데드존·ResizeObserver 재정렬). 조작부 스타일(①컨트롤 바형 ②오버레이형)은 미선택.
- 조건이 제조사마다 달라 **교차 우수 판정은 지양**하는 방향으로 수렴.

**최신 유력안 = 후보 5 "통합 보드"** (2026-07-21 저녁 기준 사용자 최종 방향, 확정 대기)
- 별도 지표란 폐지. 코멘트보드 하나에서 **지표와 메모를 함께** 씀. 쓸 때 [지표|메모] 구분.
- 지표 글 = 지표명+값+단위 + **조건 한 줄** + 작성자·날짜. **나중에 써도 항상 메모 위(상단 정렬)**.
- **제조사 소속 = 색이 아니라 위치**: 왼쪽 사진(AMAT) 아래로 AMAT 지표 열, 오른쪽(Lam) 아래로 Lam 열. 그 열의 "+ 지표" = 그 제조사 지표(선택 UI 불필요). 모바일에서도 지표 구역은 좌/우 2열 유지.
- 보드 접힘 옵션은 지표까지 숨겨서 자연 소멸 → 보드 상시 노출.
- 구현 시 DB: `demo_chat` 재활용 — 놀고 있는 `makers` 컬럼 부활(제조사 태그) + 종류 구분 컬럼(kind: metric|memo) 1개 추가. `demo_metric_defs`는 지표명 자동완성 후보로 재활용.

## 핵심 규칙 (작업 시 유지)
- 시트 매핑은 **헤더명 기반**(열 위치 비의존). 쓰기는 **관리자 인증 필수**(author=로그인이름/key=비번, 비번 재입력 없음).
- 타임라인 재파생은 `src/pages/Equipment/timeline.ts`의 `buildTimelines` **단일 창구**(STEP16 리사이즈도 재사용).
- CRUD 패턴 통일: 백엔드 Code.gs create/update/delete + LockService, 프런트 모달(body 포털) + 삭제 확인 Dialog + Snackbar + 성공 후 재fetch.
- MUI v9 → `Stack` 금지(Box flex). 색은 `tokens.ts`/StatusKind만(하드코딩 금지). 디자인/레이아웃 임의 변경 금지.
- 백엔드 배포: `npm run deploy:backend`(clasp push+redeploy). 프런트 배포 = `git push`(자동).
