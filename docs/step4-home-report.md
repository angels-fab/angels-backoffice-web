# STEP 4 — Dashboard(Home) 리디자인 (완료 보고)

> 작업일: 2026-06-13
> 첫 실제 페이지 리디자인. Home을 "소개/메뉴 진입" 화면에서 **연구센터 운영 대시보드**로 전환.

---

## 1) 기존 구조 문제점

- **소개 페이지 성격**: 최상단이 환영 배너(`Greeting`) + FAB 구축 로드맵 hero(76px 아이콘). 운영 정보가 아래로 밀림.
- **정보 위계 역전**: 참고 정보인 로드맵이 1순위, 정작 **오늘 일정·진행중 업무**는 작은 벤토 카드 미리보기(3건)로만 노출.
- **5초 파악 불가**: 진입 시 오늘 무엇을 해야 하는지 한눈에 안 들어옴.
- **레이아웃 제각각**: 홈 전용 `bento-grid`(max-width 1180, gap 14, padding 36) — 다른 페이지(1320)와 불일치(STEP3 인벤토리 확인).

## 2) 새로운 Dashboard 정보구조

진입 5초 내 핵심 파악을 목표로 위계 재구성:

```
① KPI Overview        전체장비 · 운영중 · 진행중업무 · 오늘일정 · 신규공지 · 이번달도입
② 오늘 일정 / 다가오는 일정   (좌: 오늘 전체 · 우: 향후 7일)   ← 중요도 최상
③ 업무 현황            진행중 · 예정 · 완료 · 지연 (상태별 집계)
④ 장비 현황            운영중 · 설치중 · 도입예정 · 비가동 → /equipment
⑤ 공지사항             최근 5건 → 클릭 시 Drawer 상세
⑥ FAB 구축 로드맵       하단 참고 영역으로 강등
```

## 3) 적용된 Layout System 요소 (STEP 3)

- `PageContainer` — 폭 1400 · 상단 32 · 좌우 24 (홈 전용 `<main>` 제거 → 이중 패딩 해소)
- `PageHeader` — 아이콘 + "운영 대시보드" + 업데이트 시간
- `ContentSection` ×6 — 섹션 간격 24 통일
- `CardGrid` — KPI 6열 / 상태 4열, 간격 16, 반응형(Mobile 1 · Tablet 2)

## 4) 적용된 Design System 요소 (STEP 1·2)

- `KpiCard` ×6 (숫자 강조)
- `StatusChip` — 업무·장비·공지 상태 색 일관 표시
- `AppCard` — 섹션 표면 / 일정·공지 리스트
- `AppDrawer` — 공지 상세(본문 DOMPurify 살균, 우측 슬라이드)
- `EmptyState` — 일정·공지 없음
- `SectionHeader` — 섹션 제목 + 전체보기
- 색은 전부 테마 토큰/데이터 기반(하드코딩 제거)

## 6) 화면 확인 & 데이터 주의

- 확인: `http://localhost:3600/` (dev, HMR 반영) — 자동 캡처는 브라우저 미연동으로 생략
- **데이터 주의(투명 공개)**:
  - 캘린더 이벤트에 **담당자 필드가 없어** 일정은 시간·유형·장소로 표시(명세의 "담당자"는 데이터 미보유)
  - 업무 상태는 시트에 명시 컬럼이 없어 **파생**: 완료=완료일자 입력 / 진행중=진행중체크(share) / 예정=시작일자 미래 / 지연=그 외 미완료. 시트 운영 방식과 다르면 알려주시면 조정
  - "이번달 도입" = `installDate`가 이번 달인 장비 수

## 검증

- `type-check`·`build` 통과 ✅ / 다른 6개 페이지·셸 **미수정** ✅

- 후속: `Greeting.tsx`·`previews.tsx` 등 죽은 코드 정리 과제 분리(별도 진행)
