# STEP 6 — 업무현황 Command Center (완료 보고)

> 작업일: 2026-06-14
> "업무 목록 페이지" → **업무 지휘판(Command Center)**. 무엇이 급한지·누가 바쁜지·곧 끝낼 일이 한눈에.

---

## 데이터 전제 (중요)

업무 시트엔 **'예정/완료/지연' 상태나 '마감일' 컬럼이 없다.** 실제 상태는 플래그:
- 진행중 = `share`(J열) · Remind = `!share && remind`(K열) · 지난 = `!share && !remind` · 센터장 Check = `chief`(L열, 직교)
- 날짜는 `시작일자(발의일자)`, `완료일자`뿐.

사용자 결정에 따라 **"데이터 충실형"**으로 구현(없는 상태/필드를 만들지 않음). 스키마·연동·필터·상세 데이터 모두 유지.

---

## 1) 변경 내용 (섹션 순서)

| 순서 | 섹션 | 내용 |
|------|------|------|
| ① | KPI + 상태 비율 | 진행중/지난/Remind/센터장Check 카운트(StatTile, 탭 겸용) + **상태 비율 막대(RatioBar: 진행중/Remind/지난)** |
| ② | 긴급 업무 | 센터장 Check + Remind 상위 5건, 카드, 클릭→Drawer |
| ③ | 담당자 현황 | 담당자별 카드(진행중/Remind/센터장 카운트) + **"가장 바쁜 담당자"** 하이라이트 |
| ④ | 전체 업무 목록 | 최하단으로 이동. 탭(4) + 구분/담당자 필터 + 검색 유지, 클릭→Drawer |

> **'이번주 마감' 섹션은 보류**: 시트에 실제 '마감일' 컬럼이 없어, 마감 기반 기능은 마감일 컬럼 추가 후 **STEP10 이후**에 구현(사용자 지시 2026-06-14).

상세는 **AppDrawer**: 제목·상태(StatusChip)·담당자·발의일자·완료일자·업무내용(불릿 파싱) + 관련자료 링크.

## 3) 모바일 대응 (375 / 545)

- 모든 카드 그리드 `CardGrid`: xs 1열 / 폭기준 auto-fill → **375·545px에서 카드 1열**(긴급·마감), KPI는 xs 1열
- 전체 목록 행은 `flexWrap`로 줄바꿈 → **가로 스크롤 없음**
- 섹션 순서 동일(KPI→긴급→이번주마감→담당자→전체목록)
- (자동 캡처는 브라우저 미연동으로 생략 — 반응형 로직으로 보장, 검증 워크플로 점검)
