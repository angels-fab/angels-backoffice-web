# STEP 7.5 — 장비운영관리 재해석: "구축 준비 현황판" (완료 보고)

> 작업일: 2026-06-14

---

## 왜 수정했나 (구축 단계 FAB 관점)

현재 시트 데이터는 **운영중 0 · 설치중 0 · 비가동 0 · 도입예정 29**.
즉 이 센터는 아직 **장비를 운영하는 단계가 아니라 구축(도입 준비) 단계**다.

그래서 STEP7의 "운영 Command Center"는:
- "주목 장비"가 결국 전부 도입예정 → 장비도입관리와 똑같아 보임
- "운영 상태 확인" 목적의 문구가 현실(구축 중)과 안 맞음

→ 페이지 목적을 **"운영 상태 확인" → "FAB 구축 준비 현황 파악"**으로 재정의하고,
장비도입관리("장비 개별 관리")와 역할을 분리(여긴 **"구축 진행 요약"**)했다.

## 항목별 처리

1. **섹션명**: "주목 장비" → **"도입 우선 장비"** (구축 단계에선 '주목'보다 '도입 우선'이 자연스럽고, 도입금액 순으로 우선순위 부여)
2. **카드 강화**: 담당자·장비종류·도입금액·관리번호 추가 → 도입관리 맥락의 의미 부여
3. **KPI 해석**: 단계 라벨(가동 단계/설치 단계/구축 준비중/점검 필요) + 섹션 설명 "현재 전 장비 도입예정 — FAB 구축 준비 단계"
4. **"도입 예산 TOP5/고가 장비"**: 별도 섹션 신설 대신 **"도입 우선 장비"를 도입금액 큰 순 정렬**로 흡수(동일 데이터 중복 방지). 카드에 도입금액 표시로 '고가 장비' 의도 충족.
5. **카테고리 현황**: 카테고리별 **총 도입예산(천원)** 추가
6. **목적 재정의**: 부제·섹션명·설명 문구를 '구축 준비' 관점으로 통일
