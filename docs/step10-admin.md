# STEP 10 — Guest / Admin 모드 (완료 보고)

> 작업일: 2026-06-14
> 모든 사용자 동일 화면 → **Guest(조회) / Admin(작성·관리)** 2모드. **보안 인증이 아닌 내부용 UI 게이트**(Google Sheet 기반 유지).

---

## 권한 모델

| 모드 | 권한 |
|------|------|
| Guest (기본) | 모든 페이지 **조회**, 작성·관리 버튼 숨김 |
| Admin | 작성·관리 기능 (현재: 공지 새 글쓰기) |

## 인증 — 사번 + 비밀번호 (담당자 시트, 헤더 자동 인식)

- 로그인 = **사번 + 비밀번호**. `담당자` 시트에서 대조하되, **열 위치는 헤더명으로 자동 인식**(`findManagerCols_`):
  - 사번 ← `사번`/`사원번호`/`직원번호`
  - 비밀번호 ← `비밀번호`/`패스워드`/`password`
  - 이름 ← `이름`/`성명`/`담당자`
  - → **사번 열을 다른 열로 옮겨도 자동으로 찾음.**
- 검증은 **백엔드(Apps Script)에서만**(`verifyAdmin` → `loginByEmp_`). 비밀번호는 프런트로 내려오지 않음.
- 성공 시 담당자 **이름**을 받아 `localStorage(role/adminUser/adminKey)`에 보관.

## 공지 새 글쓰기

- **Admin일 때만** "새 공지" 버튼 노출.
- **비밀번호 재입력 없음** — 로그인 시 검증된 게시자(이름)·비번을 그대로 사용해 게시. 폼에는 게시자(읽기전용)만 표시.

## 구현

| 영역 | 파일 |
|------|------|
| 권한 상태 | `auth/role.tsx` — login(사번,비번)→이름 보관, user·authKey 노출 |
| 검증 | `api/sheets.ts` `verifyAdmin(empNo,pw)→{valid,name}` · Code.gs `findManagerCols_`/`loginByEmp_`/`authError_`(헤더 기반) |
| 로그인 UI | TopBar 우상단 관리자 버튼 → `AdminLoginDialog`(사번+비밀번호) |
| 작성 | `NoticeWrite` — 게시자·비번 입력 제거, 로그인 정보 자동 사용 |
| 설정 | `/settings`(권한·버전·데이터소스) + 사이드바 메뉴 |

## 배포 & 검증

- 백엔드 **배포 완료**(clasp @33). 엔드포인트 확인:
  - `?authors=1` → 이름 목록 반환(헤더 자동 인식 성공)
  - `verifyAdmin`(더미) → `{valid:false, name:""}` 정상
- `type-check`·`build` 통과 ✅

## 사용 방법
1. 우측 상단 **관리자 모드** → **사번 + 비밀번호**(담당자 시트) 입력 → Admin 전환("관리자 · 이름")
2. 공지사항에서 **새 공지** 버튼 → 비번 재입력 없이 작성

## 비고
- 현재 Admin 전용 = 공지 새 글쓰기뿐. 향후 수정·삭제 등은 만들 때 `isAdmin`으로 게이트.
- 내부용 UI 게이트(실제 인증 아님). 비밀번호는 localStorage에 보관(재입력 생략용) — 내부 도구 전제.
- 프런트(STEP10)는 아직 로컬만 — 실사이트 반영하려면 git 코드 push 필요(백엔드는 이미 배포됨).
