# STEP 8 설계안 — 장비도입관리: Project Tracking Dashboard (제안)

> 상태: **제안(미구현)**. 장비도입관리(`/equipment`) = "장비 도입 프로젝트 관리". 운영 현황이 아니라 **"언제 도입되는가"**가 핵심.
> 성격: Procurement / Project PM 도구 (Gantt · Pipeline · Milestone). Google Calendar 스타일 지양.

---

## 1. 데이터 (이미 존재 — 새 데이터 없음)

장비도입관리 시트 = 구매 절차별 도입 단계/기간. 코드에 이미 매핑돼 있음:

- `EqRawItem.timeline: string[]` — **반월(전·후반) 단위 단계 코드** 배열
- `eq.months: TlMonth[]` — 타임라인 시간축(반월 단위, 앞뒤 빈 달 제거)
- 단계 코드(`gantt.tsx`): **사**=사전규격 · **공**=구매공고 · **평**=기술평가 · **협**=기술협상 · **제**=장비제작 · **설**=장비설치
- 그룹: `EqGroup.timeline`(장비명 대표 타임라인), 담당자 `mgr`, 도입금액 `price`

> 현재 `/equipment`는 이 데이터를 **간트차트**로 그리는 레거시 페이지. STEP8은 이를 PM 대시보드로 재구성.

---

## 2. 섹션 구성 (제안)

| 순서 | 섹션 | 내용 |
|------|------|------|
| ① | **도입 진행 개요(KPI)** | 전체 장비 수 · 진행중 · 완료(설치 도달) · 착수 전 — StatTile |
| ② | **단계 파이프라인** | 사전규격→구매공고→기술평가→기술협상→장비제작→장비설치 각 단계의 **현재 장비 수** (RatioBar 또는 단계별 StatTile 행) = Acquisition Pipeline |
| ③ | **병목 · 지연** | 병목=가장 많이 정체된 단계 / 지연=현재 반월 기준 예정 구간을 지난 장비(타임라인 위치로 파생) — 카드, 클릭→Drawer |
| ④ | **담당자별 진행** | 담당자별 장비 수 + 현재 단계 분포 |
| ⑤ | **도입 타임라인(Gantt)** | 장비별 행 × 월 축 × 단계 색 막대. 기존 gantt 로직 재사용 + 디자인 시스템 셸로 재단장. 장비 클릭→Drawer. 현재 반월 표시선 추가 |
| ⑥ | **상세 Drawer** | 장비명 · 담당자 · 현재 단계 · 단계별 기간 · 총소요기간 · 도입금액 |

---

## 3. 디자인 / 컴포넌트

- 디자인 시스템 그대로: `PageContainer · ContentSection · AppCard · StatTile · RatioBar · StatusChip · AppDrawer · CardGrid`.
- 단계 색: 기존 `TL_STAGE_COLOR`(6단계)를 accent 토큰으로 정렬해 통일(StatusChip/RatioBar와 일치).
- **간트**는 특수 시각화라 전용 그리드 유지(FullCalendar처럼) + 셸/색만 디자인 시스템화. 새 디자인 언어 금지.
- 신규 파일(예상): `pages/Equipment/stageMeta.ts`(단계 코드↔라벨↔색↔진행도), `EqProjectDrawer.tsx`, 섹션 컴포넌트.

---

## 4. 핵심 파생 로직 (제안)

- **현재 단계**: `timeline` 배열에서 가장 마지막(또는 현재 반월 이전 마지막) 단계 코드.
- **진행률**: 단계 순서(사→공→평→협→제→설) 기준 (현재 단계 index / 6).
- **완료**: timeline에 '설'(장비설치) 도달.
- **지연(파생)**: 장비의 timeline 예정 구간이 현재 반월보다 앞에서 끝났는데 '설' 미도달 → 지연 의심.

## 5. 확인 필요 (데이터 질문 — 빌드 전 확정)

1. **"총소요기간"**이 시트의 별도 컬럼인지, timeline 길이에서 계산하는지?
2. **지연** 판정 기준 — 시트에 '목표 도입일'이 있으면 그것 기준, 없으면 위 timeline 파생으로 진행?
3. 간트는 **그룹(장비명) 단위**로 그릴지, **개별 장비(raw)** 단위로 그릴지? (현재 레거시는 그룹 기준 추정)
4. 단계 코드 6종(사/공/평/협/제/설) 외에 시트에 다른 코드가 있는지?

→ 위 4가지 확정 후 STEP8 구현 착수. (운영관리와 중복 없이 "언제/어느 단계까지" 중심으로 구성)
