# STEP28 — 개선제안 페이지 신설 (소통 그룹)

> 2026-06-19 · 프런트 + 백엔드. 백엔드 clasp 배포 **@43**(URL 불변), 프런트 `main` 자동배포. type-check·build 통과, 라이브 dev 검증(콘솔0, 실데이터 4건).

사용자가 홈페이지 개선 의견을 남기고 관리자(담당자)가 반영 상태를 관리하는 게시판.

## 데이터 (구글시트 '개선사항' 시트)
- 헤더 **3행**(자동 탐지), 실제 헤더: `번호·유형·개선위치·긴급·제목·관련자료·작성자·제안일자·개선내용·담당자·상태·완료일자·사유`
- 백엔드는 헤더명(+동의어)으로 인식 → 열 순서 비의존. `사유`는 불가·보류 **공용 열**.

## 백엔드 (google-apps-script/Code.gs, @43)
- `improveCtx_()` — '개선사항' 시트 헤더 자동 탐지(첫 10행에서 '제목' 기준) + 동의어 col 맵.
- `getImprovements_()` (GET `?action=getImprovements`) — 헤더명 기준 객체 배열(+headers) 반환. 조회 인증 불필요.
- `createImprovement_()` (POST) — 로그인 사용자(담당자 시트 인증)만. 상태=접수중, 제안일자=오늘(미지정 시), 번호 자동(max+1), 긴급 체크박스.
- `updateImprovement_()` (POST) — **담당자만**(행의 담당자 == author) 상태 변경. 개선완료→완료일자 자동(상태 바뀌면 비움), 반려/보류→사유 저장(그 외 비움).

## 프런트
- 신규: `src/pages/Improve/{index,ImproveWrite,improveMeta}.tsx`, `src/store/slices/improveSlice.ts`, 타입 `ImprovementItem`, API `fetchImprovements/createImprovement/updateImprovement`.
- 사이드바 **새 "소통" 그룹 + 개선제안**(`LightbulbOutlined`), 배지=접수중 건수(`useNavBadges.improve`). 라우트 `/improve`. MainLayout 진입 시 `loadImproveData`.
- **목록형 표**: 열 `번호·유형·개선위치·제목·작성자·제안일자·담당자·상태·비고`(제목·비고만 좌측, 나머지+헤더 가운데). 최신순 정렬.
  - 제목칸: 긴급이면 맨 앞 `PriorityHigh`(빨강), 관련자료 있으면 끝에 외부링크 아이콘(없으면 미표시).
  - 행 클릭 → 개선내용 **아코디언**(헤더·화살표 없음).
  - 상태칸: 담당자에게 드롭다운(5상태), 그 외 읽기전용 칩. 색 접수중=초록·검토중=회색·개선완료=파랑·보류=주황·반려=빨강.
  - 비고칸: 개선완료→완료일자 / 반려·보류→사유(담당자는 **인라인 편집**, 상태 전환 시 Enter로 저장) / 그 외→—.
- **필터 탭**: 접수중·검토중·개선완료·반려·보류(전체 탭 없음). 기본 접수중. 같은 탭 재클릭=전체. **Shift+클릭=중복선택**. 탭 색=상태 색.
- **제안 등록**(관리자): 모달(긴급·유형·개선위치·제목·개선내용·관련자료·담당자). 작성자=로그인 이름 자동, 담당자=작성자가 선택(상태변경 권한자).

## 권한
- 작성 = 로그인 사용자. 상태변경 = 담당자(프런트 게이팅 + 백엔드 검증).
